# PLC Ladder Viewer — GitLab Web IDE Extension

Renders Rockwell Allen-Bradley `.l5x` files as visual ladder logic diagrams inside GitLab Web IDE (and VS Code). Supports AOI definitions, standard function blocks, contacts, coils, and parallel branches — styled to match Studio 5000.

---

## Features

- **View mode** — click any `.l5x` file in the file tree → opens as ladder diagram
- **Toolbar button** — open viewer or trigger diff from the editor title bar
- **Diff mode** — compare two git refs side-by-side *(stub ready, wired in next phase)*
- Handles: `cPhotoEye` and other lowercase AOIs, `BST/NXB/BND` branches, `[path,path]` bracket notation, NOP, all standard Rockwell instructions

## Local development setup

**Prerequisites:** Node.js 18+, npm, VS Code

```bash
git clone https://github.com/01nodo/diff_tool.git
cd diff_tool
npm install
npm run build
```

Press **F5** in VS Code to launch the Extension Development Host with the extension loaded. Open any `.l5x` file to test.

---

## Build & package

```bash
npm run build      # compiles everything
npm run package    # → diff_tool-0.1.0.vsix
```

---

## Install on self-hosted GitLab

1. Build the `.vsix` on any machine with Node 18+
2. In your GitLab instance: **Admin Area → Web IDE → Extensions**
3. Upload `diff_tool-0.1.0.vsix`
4. The extension is now available to all users — no per-user install needed

## How it works

```
User clicks .l5x
       │
       ▼
extension.ts (Node/VS Code host)
  registerCustomEditorProvider("plc.ladderViewer")
  reads file → vscode.workspace.fs.readFile()
  creates WebviewPanel
  postMessage({ type: "load", xml })
       │
       ▼
media/viewer.js (React, runs in WebviewPanel)
  main.tsx receives message → passes xml prop to App.jsx
  App.jsx: parseL5X() → parseRung() → SVG render
```

## Diff mode (next steps)

`extension.ts` already has `plc.openLadderDiff` registered. It:
1. Prompts for two git refs (branch names, SHAs, tags)
2. Fetches file content at both refs via the VS Code git extension API
3. Posts `{ type: "diff", xmlA, xmlB, refA, refB }` to the webview

To complete diff mode, add semantic rung-matching and red/green highlighting to `App.jsx` when it receives both `initialXml` and `diffXml` props.
